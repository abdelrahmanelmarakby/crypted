# Screenshot Capture Guide for App Stores

This guide will help you capture the perfect screenshots for both Apple App Store and Google Play Store.

   - Optional

3. **Tablet 10-inch**
   - Resolution: 2048 x 1536 pixels

## ğŸ”§ Post-Processing

### Recommended Tools
- **Figma** - Design screenshot templates
- **Sketch** - Create device mockups
- **Adobe Photoshop** - Advanced editing
- **Canva** - Quick templates (Pro account)
- **Screenshot.rocks** - Automated device frames
- **Fastlane Frameit** - Automated screenshot framing

### Processing Steps
1. **Crop** to exact dimensions required
2. **Color Correct** if needed (brightness, contrast)
3. **Remove** any sensitive data
4. **Add** device frames (optional)
5. **Add** captions (optional)
6. **Optimize** file size (reduce to <1MB if possible)
7. **Verify** dimensions and format
8. **Name** systematically (e.g., `01_chat_list_iphone67.png`)

### Batch Processing Script (ImageMagick)
```bash
#!/bin/bash
# Resize all screenshots to exact dimensions

# iOS iPhone 6.7"
for file in ios_screenshots/*.png; do
  convert "$file" -resize 1290x2796! -quality 90 "processed/$file"
done

# Android Phone
for file in android_screenshots/*.png; do
  convert "$file" -resize 1080x1920! -quality 90 "processed/$file"
done
```

## ğŸ¬ Video Preview Tips

### iOS App Preview
**Specs:**
- Length: 15-30 seconds (maximum 30 seconds)
- Resolution: Same as screenshot requirements
- Format: .mov, .m4v, or .mp4
- Codec: H.264 or HEVC
- Audio: Optional (AAC)

**Content:**
1. Open with app icon animation (2s)
2. Show 3-4 key features (20s)
3. End with call-to-action (3s)

### Android Promo Video
**Specs:**
- Length: 30 seconds - 2 minutes
- Aspect Ratio: 16:9
- Upload to YouTube
- Link in Play Store Console

**Recommended Structure:**
- 0-5s: Hook (problem or exciting feature)
- 5-45s: Feature demonstration
- 45-55s: Social proof or testimonial
- 55-60s: Call to action
